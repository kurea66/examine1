カプセル化

ヒューマンエラーを防ぎ安全なシステムを作るための仕組み

アクセス制御(access control)・・・大切な情報に制限を加えること

４つのアクセス制御レベル

アクセス修飾子

public・・・どのクラスからもアクセス化。

protected・・・自分と同じパッケージに属するか、自分を継承した子クラス

package private(何も書かない)・・・自分と同じパッケージに属するクラス

private・・・自分のクラスのみ

getterとsetter

getter・・・プライベートのフィールドの値を返すだけのメソッド

ex:public フィールドの型 getフィールド名(){return this.フィールド名;}

setter・・・指定された値を代入するだけのメソッド

public void setフィールド名(フィールドの型　任意の変数名){this.フィールド名=任意の変数名;}

アクセサ(accessor)・・・getter,setterの総称

setterメソッド内で設定された値の妥当性をチェック

クラスに対するアクセス制御

package private(クラス宣言の際に何も書かない)・・・同一パッケージからしかそのクラスにアクセスできない

例外（エラー）を投げる

throw new IllegalArgumentException("メッセージ")…エラーを発生させ、メッセージの内容を表示する。

IllegalArgumentException…Javaに元から入っているクラス。　エラー発生行を出してくれるよ。

静的メンバ

静的メンバ(static member)・・・staticキーワードがついたフィールドやメソッド

静的クラスのフィールドにアクセスする場合はクラス名.フィールド名

			public class Hero{
				static int money;

				Hero.money=100;
			}

一つのメモリ領域を直接いじる感じ

staticメソッドからはstaticメソッドにしかアクセス出来ない!インスタンスにはアクセス不可

public static final 中身…静的な定数。　どこからでも呼び出せる。Math.PI等はMathクラスの静的フィールドのPIを呼び出している。

単語、関数等

Illegal…違法の、不法の

Arguments…引数、論拠、議論、主張

exception…例外、除外、異例、異議
