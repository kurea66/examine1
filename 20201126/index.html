Java

部品化

部品化…複数のソースファイルを分けて開発すること。

部品化したメソッドを呼び出すには,クラス名(file名).メソッド名

ただし同じ階層にないと無条件には読み込んでくれない

JARファイル・・・JavaARchiveの略。　複数のクラスファイルを１つにまとめるファイル形式

パッケージ

パッケージにするにはソースコードの先頭にpackage文を記述する！

先頭に　package calc.logics;と記述する。パッケージ名は小文字、.で区切っても可

デフォルトパッケージ（無名パッケージ)・・・パッケージ文がないソースコード

FQCN(full qualified class name),完全限定クラス名や完全修飾クラス名パッケージ名.クラス名で呼び出せる。
ex: calcapp.logics.CalcLogit.tasu等

パッケージに入れたクラスはFQCNでないと起動できない！java file名では起動できない！！

クラスローダー・・・完全限定名を指定されたら、その名前を持つクラスのクラスファイルをPC内から検索し利用可能にする役割。

クラスパス…クラスローダーがクラスファイルを探す際に探す場所を指定。

クラスパスの指定…起動時に指定java -cp フォルダ名　ファイル名

環境変数にクラスパスを設定

特に指定しない…現在いるフォルダを参照

クラスパスで指定できる対象

クラスファイルが置かれているフォルダの場所を指定するもの。 ex: c:\workと指定するとworkフォルダ内のクラスファイルが検索対象となる

クラスファイルが入ったJARファイルやZIPファイルを指定
例えばCalc.classが入ったcalcapp.jarといるファイルがc:\work\jarsにある場合c:\work\jar\calcapp.jarというクラスパスを指定することで読み込める

複数のフォルダ、JAR/ZIPファイル、それらの組み合わせを指定
デリミタ文字で区切ってクラスパスを指定。,デリミタ文字はWindowsは「;」、mac,Linuxは「:」
ex:c:\work; c:\work\jars.app

インターネットドメインを前後逆にしたパッケージ名を用いることを推奨されている
ex: foo.example.comの場合com.example.foo.パッケージ名

java -verbose ファイル名で使用しているすべての完全限定クラスが表示される

java.langクラスはimportしなくても最初からimportされている

関数、タグ等

mkdir -p a/b/c…aというディレクトリの中にbというディレクトリ、その中にcというディレクトリを作成するときのオプション

Thread.sleep(ミリ秒);…ミリ秒間一時停止させる。
