オブジェクト指向

今までやってきたのは手続き型プログラミング

オブジェクトは属性と操作を持っている。

フィールド･･･パラメーター(属性)の事をjavaではフィールドと呼ぶ

オブジェクト指向の中のメソッドはほぼstaticがつかない

		public class Hero{
			String name;
			int hp;

			public void sleep(){
				this.hp = 100;
				System.out.println(this.name + "は、眠って回復した!");
			}
		}

this.hpのthisは自分のインスタンスを表すワード。インスタンスのhpを100にするということ

		Hero h=new Hero();
		h.name="ロト";
		h.sleep();

作ったメソッドが型になる。今回はHero型のhという変数のnewでHeroメソッドから読み込んで作成している

コンストラクタ

newする際に名前に仮引数を渡せるようにする。渡した値をそのまま代入する。

コンストラクタのルール…コンストラクタ名は必ずクラス名と同じにすること

記述例

	
		public class Hero{
			String name;
			int hp=100;
			public Hero(String name){
				this.name=name;
			}
		}

コンストラクタを作成すると指定した引数をnewと同時に代入しないと作れない！！

コンストラクタで引数無くても作れるようにするのであればコンストラクタをオーバーロードして、引数の無いものを作る

コンストラクタを作るか作らないかは設計者の意図。絶対に入れなければいけないのか、あるほうが使いやすいのかないほうが使いやすいのかよく考えて作成すること。

デフォルトコンストラクタ

そのクラスにコンストラクターを一つも書かなかった場合、何もないコンストラクターが自動で用意される

コンストラクタの中で別のコンストラクタを呼び出すには1回しか呼び出せない。

this(引数)…同一クラス内のコンストラクタの呼び出し。先頭行に書かないとコンパイルエラー！！

オブジェクト指向の三大機能

継承・多態性・カプセル化

単語、関数、タグ等

インスタンス・・・設計区間からメモリ区間に呼び出されたものをインスタンスという

インスタンス化・・・クラスからインスタンスを生成すること。
